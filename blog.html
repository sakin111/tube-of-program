what is the scope of var, let, and const?

var

var is a global scope Variable declaration method, where the variable can be redeclare many time.it is also a fact that it is hoist to the top of their containing function or global scope. This means they are processed before the code is executed, but their assignment is not hoisted. here is a example given below:

function example() {
var x = 10;
if (true) {
var x = 20; // This reassigns the outer 'x'
}
console.log(x); // Outputs 20
}
let

Variables declared with let have block scope, which means they are only accessible within the block, in which they are defined.let variables are not hoisted to the top of the block like var do.(let) can not be redeclared in the same function.here is a example given below:

function example() {
let x = 10;
if (true) {
let x = 20; // This creates a new 'x' within the block scope
}
console.log(x); // Outputs 10 (the outer 'x' is not affected)
}
const

const variables are not hoisted to the top of the block, and they are only accessible after they are defined. the const variables are immutable and cannot be reassigned after they are initialized. here is a example given below:

function example() {
const x = 10;
if (true) {
const x = 20; // This creates a new 'x' within the block scope
}
console.log(x); // Outputs 10 (the outer 'x' is not affected)
}

Tell us the use cases of null and undefined ?

null

It is common to use null when you want to clear or reset the value of an object or variable that previously held an object reference. Developers can explicitly assign the value null to a variable or property to indicate that it has no value or should be considered empty. here is a example given below:

let y = null; // y has no value
function clearValue(obj) {
obj = null; // Explicitly set obj to null to indicate it's empty
}
const myObject = { data: 'some data' };
clearValue(myObject);
console.log(myObject); // { data: 'some data' } - myObject is still defined,
but its value is null
undefined

Variables that are declared but not initialized or properties that do not exist in an object have the value undefined.If a function is called with some arguments than declared parameters, the missing parameters will have the value undefined.Developers may use undefined to indicate that a variable or property is intentionally left empty or uninitialized. here is a example given below:

let x; // x is undefined
function example(param) {
console.log(param); // If called with no argument, param is undefined
}
const obj = {};
console.log(obj.property); // obj.property is undefined

What's an API and REST API?

API is an application programming interface that contract between an information provider and an information userâ€”establishing the content required from the consumer (the call) and the content required by the producer (the response). For example, the API design for a weather service could specify that the user supply a zip code and that the producer reply with a 2-part answer, the first being the high temperature, and the second being the low. In other words, if you want to interact with a computer or system to retrieve information or perform a function, an API helps you communicate what you want to that system so it can understand and fulfill the request.

RESTFUL API is an interface that two computer systems use to exchange information securely over the internet. Most business applications have to communicate with other internal and third-party applications to perform various tasks. For example, to generate monthly payslips, your internal accounts system has to share data with your customer's banking system to automate invoicing and communicate with an internal time-sheet application. RESTful APIs support this information exchange because they follow secure, reliable, and efficient software communication standards.
